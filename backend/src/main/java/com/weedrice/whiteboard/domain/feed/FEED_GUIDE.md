# Feed 도메인 가이드

`feed` 도메인은 각 사용자에게 맞춤화된 콘텐츠 목록(피드)을 제공하는 기능을 담당합니다.

## 1. 주요 기능 및 로직

`FeedService`를 통해 다음과 같은 핵심 비즈니스 로직을 처리합니다.

- **피드 조회 (`getUserFeeds`):**
  - 현재 로그인한 사용자의 피드 목록을 최신순으로 페이징하여 조회합니다.
  - `user_feeds` 테이블에 미리 생성된 데이터를 기반으로 합니다.

- **피드 생성 (`generateFeeds`):**
  - **(미구현)** 이 기능은 주기적으로 실행되는 배치 작업을 통해 구현되어야 합니다.
  - 각 사용자별로 다음 정보를 수집하여 `user_feeds` 테이블에 저장하는 로직이 필요합니다.
    - 구독한 게시판의 새 글
    - 관심 태그가 달린 새 글
    - 팔로우하는 사용자의 활동 (글 작성, 댓글 등)

## 2. API Endpoints

`FeedController`에서 다음 API를 제공합니다.

| Method | URI                       | 설명           |
| :----- | :------------------------ | :------------- |
| `GET`    | `/api/v1/users/me/feeds`  | 내 피드 목록 조회 |

## 3. 관련 DB 테이블

`feed` 도메인은 주로 다음 테이블과 상호작용합니다.

| 테이블명      | 엔티티      | 설명                     |
| :------------ | :---------- | :----------------------- |
| `user_feeds`  | `UserFeed`  | 사용자별 맞춤 피드 정보  |

이 문서는 `feed` 도메인의 기능을 이해하고 개발하는 데 도움을 주기 위해 작성되었습니다.
