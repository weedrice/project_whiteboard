# Report 도메인 가이드

`report` 도메인은 사용자/게시글/댓글 등에 대한 신고 접수와 관리(처리 상태 변경)를 담당합니다.

## 1. 주요 기능 및 로직
- 신고 생성: 동일 사용자가 동일 대상에 중복 신고하지 못하도록 검증 후 신고 저장.
- 신고 조회: 슈퍼관리자가 상태/대상 유형별로 신고 목록을 조회.
- 내 신고 조회: 신고자가 본인이 올린 신고 목록을 확인.
- 신고 처리: 슈퍼관리자가 처리자(Admin) 정보를 함께 기록하며 상태/비고를 업데이트.

## 2. API Endpoints

| Method | URI | 설명 |
| :----- | :---------------------------------------- | :----------------------------- |
| `POST` | `/api/v1/reports/users` | 사용자 신고 생성 |
| `POST` | `/api/v1/reports/posts` | 게시글 신고 생성 |
| `POST` | `/api/v1/reports/comments` | 댓글 신고 생성 |
| `POST` | `/api/v1/reports` | 범용 신고 생성(타입/사유 전달) |
| `GET` | `/api/v1/reports/me` | 내가 신고한 내역 조회 |
| `GET` | `/api/v1/admin/reports` | 신고 목록 조회(필터 포함) |
| `PUT` | `/api/v1/admin/reports/{reportId}` | 신고 처리/상태 변경 |

## 3. 관련 DB 테이블

| 테이블명 | 엔티티 | 설명 |
| :------- | :----- | :--- |
| `reports` | `Report` | 신고 대상/사유/상태/처리자 기록 |
